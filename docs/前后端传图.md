前端 → 后端：图片怎么传过去

  用户选图 → FileReader读成base64 → JSON body → POST请求 → 后端接收

  单张流程 POST /api/v1/detect

  1. 用户选择图片（File对象）
  2. FileReader.readAsDataURL(file)  → 得到 "data:image/jpeg;base64,/9j/4AAQ..."
  3. 截取逗号后面的部分作为纯base64字符串
  4. 发送JSON请求：

  {
    "image": {
      "type": "base64",
      "data": "/9j/4AAQSkZJRg..."    // ← 这就是图片的二进制数据，用base64编码成字符串
    },
    "confidence": 0.5
  }

  批量流程 POST /api/v1/detect/batch

  1. 用户选择多张图片（File[]）
  2. Promise.all 并发把每张图都读成base64
  3. 打包成一个JSON请求：

  {
    "images": [
      { "type": "base64", "data": "/9j/4AAQ..." },   // 第1张
      { "type": "base64", "data": "iVBORw0KGgo..." },  // 第2张
      { "type": "base64", "data": "R0lGODlh..." }      // 第3张
    ],
    "confidence": 0.5
  }

  后端收到后做什么

  后端(Spring Boot)                          Python推理服务(8000)
      │                                            │
      ├─ 收到 base64 字符串                         │
      ├─ 解码成字节数组                              │
      ├─ 保存为临时 .jpg 文件                        │
      ├─ POST /api/v1/detect ─────────────────────→ │
      │                                            ├─ 加载YOLO病害模型
      │                                            ├─ 加载YOLO虫害模型
      │                                            ├─ 执行推理
      │                                            ├─ 画标注框
      │                                            └─ 返回结果JSON
      ├─ ←───────────────────────────────────────── │
      ├─ 返回给前端                                  │
      └─ 清理临时文件                                │

  前端直接用 annotated_url 作为 <img :src="url"> 显示标注图。

  关键点

  - base64 编码会让图片体积增大约 33%（640x640 的 JPEG 约 50KB → base64 约 67KB），20 张图并发请求体约 1.3MB，WiFi 环境完全没问题
  - 批量接口是单次 HTTP 请求，不是逐张发，后端内部按顺序推理
  - 标注图通过 MinIO 对象存储提供访问 URL，无需 base64 传输
