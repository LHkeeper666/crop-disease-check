## 一、认证模块 (Auth)

### 1. 登录接口未对接 ❌

| 项目 | 说明 |
|------|------|
| 前端 | `stores/auth.ts` login() 使用 **mock 数据**，未调用后端 |
| 后端 | `POST /api/auth/login` 已实现 |
| 前端传参 | `{username, password}` |
| 后端期望 | `{username, password}` (LoginDTO) |
| 响应格式 | `{code, data: {token, refreshToken, expiresIn, userInfo}}` |

**影响：** 前端拿到的是 mock token，后续所有需要认证的接口都会失败。

---

### 2. 注册接口缺少验证码 ❌

| 项目 | 说明 |
|------|------|
| 前端 | `stores/auth.ts` register() 传 `{email, username, password}` |
| 后端 | RegisterDTO 期望 `{username, password, email, code}` |
| 缺失字段 | `code`（邮箱验证码） |

---

### 3. 退出登录未对接 ❌

| 项目 | 说明 |
|------|------|
| 前端 | `stores/auth.ts` logout() 只清除 localStorage |
| 后端 | `POST /api/auth/logout` 已实现 |
| 前端传参 | 无（应在 Header 带 Bearer token） |

## 五、Vite 代理配置 ✅

```
前端请求              代理规则                    目标服务
/api/v1/detect/*  →  http://localhost:8000    →  Python 推理服务
/api/*            →  http://localhost:8080    →  Spring Boot 后端
/proxy/deepseek/* →  https://api.deepseek.com →  DeepSeek API
```

后端 `context-path: /api`，Controller 路径如 `@RequestMapping("/auth")`，最终路径 `/api/auth/*`，与前端一致。
