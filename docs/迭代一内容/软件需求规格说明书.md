## 1. 引言

### 1.1 编写目的

本文档旨在对"智慧农业病虫害无人值守监测与决策系统"进行全面、系统的需求规格说明，明确系统的功能需求、非功能需求、外部接口、数据模型及架构约束，作为后续系统设计、开发、测试与验收的基准文档。

### 1.2 项目背景

传统农业病虫害管理系统存在以下痛点：

- **被动识别**：仅提供"拍照→识别→展示"的单向流程，缺乏自动化闭环能力
- **信息碎片化**：灾情数据以零散图表呈现，缺乏全局复盘与智能决策支持
- **专家响应滞后**：发现灾情后需人工联系专家，沟通链路长、效率低
- **人工值守成本高**：需要工作人员持续盯屏，无法实现真正的无人值守

本系统旨在构建一个**从数据感知到智能决策的工业级闭环系统**，实现异构双推理流水线、事件驱动工单、LLM 自动生成报告、邮件 Token 回调闭环等核心能力。

### 1.3 系统范围

本系统覆盖温室大棚病虫害监测的完整生命周期：

```
感知层：多机位摄像头采集 → CV 模型推理 → 空间化数据融合
   ↓
决策层：实时应急工单 + 周期性 LLM 诊断报告
   ↓
执行层：邮件外发 → 专家一键回调 → 大屏实时状态刷新
```

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| CV | Computer Vision，计算机视觉 |
| LLM | Large Language Model，大语言模型 |
| QuickSelect | 快速选择算法，用于在 O(n) 时间内找到第 k 大元素 |
| 热力图 | 以颜色深浅表示数据密度/强度的可视化图表 |
| 工单 | 系统自动生成的灾情事件处理单 |
| Token 回调 | 通过邮件中嵌入的唯一 Token 实现无登录操作的机制 |
| 状态机 | 描述工单生命周期的状态转换模型 |
| 双流水线 | 病害推理流水线与虫害推理流水线并行运行的架构 |

### 2.3 用户角色定义

| 角色 | 描述 | 主要职责 |
|------|------|----------|
| 系统管理员 | 系统的最高权限用户 | 管理摄像头配置、区域划分、用户权限、系统参数 |
| 农技专家 | 具备病虫害专业知识的技术人员 | 接收工单邮件、远程指导灾情处理、审核 AI 报告 |

### 3.2 FR-01：摄像头机位映射与网格区域配置

#### 3.2.1 功能描述

为系统提供基础的空间坐标体系，将物理大棚划分为可管理的网格区域，并将每个摄像头机位映射到对应的物理坐标和覆盖范围。

#### 3.2.2 功能详细需求

| 子编号 | 需求描述 | 验收标准 |
|--------|---------|---------|
| FR-01-01 | 支持管理员在大棚平面图上通过拖拽方式定义网格区域（如 M×N 网格） | 可在画布上绘制矩形/多边形区域，支持缩放和移动 |
| FR-01-02 | 每个网格单元格具有唯一标识（如 Grid-A1、Grid-B3），并关联物理坐标范围 | 标识唯一且持久化存储 |
| FR-01-03 | 支持为每个摄像头机位配置：名称、RTSP 地址、安装坐标、覆盖网格范围、朝向角度 | 配置信息完整且可编辑 |
| FR-01-04 | 支持网格区域的增删改查操作 | CRUD 操作均有相应 API 和前端界面 |
| FR-01-05 | 摄像头覆盖范围在平面图上可视化展示（扇形/矩形视角锥） | 视角锥与实际覆盖区域一致 |

#### 3.2.3 输入/处理/输出

- **输入**：管理员上传的大棚平面图（支持 PNG/JPG/SVG）、摄像头设备参数
- **处理**：前端 Canvas 渲染网格叠加层，后端持久化空间映射关系
- **输出**：网格区域配置数据（JSON）、摄像头-网格映射关系表

#### 3.2.4 工业/智能体现点

> 提供基础的空间坐标体系，为后续数据空间化（热力图、空间查询）打下基础。属于传统 Web 功能，但为智慧感知层提供必要的空间语义。

### 3.3 FR-02：视频流广场与图片抓拍看板

#### 3.3.1 功能描述

支持多机位实时视频流的同时播放与轮巡，以及定时/手动抓拍图片的展示。

#### 3.3.2 功能详细需求

| 子编号 | 需求描述 | 验收标准 |
|--------|---------|---------|
| FR-02-01 | 支持 4/6/9 宫格布局的多机位同时播放 | 4 宫格延迟 < 2 秒，9 宫格延迟 < 3 秒 |
| FR-02-02 | 支持手动抓拍当前帧并保存至本地/服务端 | 抓拍图片分辨率与源流一致 |
| FR-02-03 | 支持自动轮巡模式，按预设时间间隔切换机位 | 轮巡间隔可配置（5s-60s） |
| FR-02-04 | 支持单击放大某个机位至全屏播放 | 全屏模式下支持 ESC 退出 |
| FR-02-05 | 视频流支持 RTSP/HLS/FLC 协议接入 | 至少支持其中两种协议 |
| FR-02-06 | 抓拍图片带有时间戳和机位信息水印 | 水印位置可配置 |

#### 3.3.3 输入/处理/输出

- **输入**：摄像头 RTSP 流地址
- **处理**：流媒体服务器转码/转封装 → 前端 MSE/WebRTC 播放
- **输出**：实时视频画面、抓拍图片文件

### 3.4 FR-03：历史病虫害数据图表

#### 3.4.1 功能描述

以折线图、饼图、柱状图等常规 ECharts 统计图表展示过去和当下的病虫害数据，用于灾情复盘。

#### 3.4.2 功能详细需求

| 子编号 | 需求描述 | 验收标准 |
|--------|---------|---------|
| FR-03-01 | 折线图展示指定时间范围内病虫害识别次数趋势 | 支持日/周/月三个时间粒度切换 |
| FR-03-02 | 饼图展示各病虫害类型占比 | 点击扇区可下钻查看详情 |
| FR-03-03 | 柱状图展示各网格区域的受灾对比 | 支持排序和筛选 |
| FR-03-04 | 支持时间范围选择器（近 7 天/近 30 天/自定义） | 自定义范围不超过 365 天 |
| FR-03-05 | 图表数据支持导出为 CSV/PNG | 导出文件包含完整的元数据头 |

#### 3.4.3 输入/处理/输出

- **输入**：后端病虫害识别记录数据库
- **处理**：按时间/区域/类型聚合统计
- **输出**：ECharts 渲染的交互式图表

### 3.5 FR-04：2.5D 空间网格时空热力图 

#### 3.5.1 功能描述

将 CV 识别数据与空间网格结合，通过 CSS3D 变换构建 2.5D 大棚俯视图，以颜色深浅演变展示虫害扩散趋势，具备"数字孪生"雏形。

#### 3.5.2 功能详细需求

| 子编号 | 需求描述 | 验收标准 |
|--------|---------|---------|
| FR-04-01 | 基于 CSS3D Transform 构建 2.5D 等距视角的大棚俯视图 | 视角可旋转（±30°），可缩放 |
| FR-04-02 | 每个网格单元格根据该区域的病虫害置信度值映射颜色（绿→黄→红渐变） | 颜色映射符合直觉，支持色盲友好模式 |
| FR-04-03 | 支持时间轴回放功能，拖动时间轴可查看历史任意时刻的热力状态 | 时间轴粒度精确到小时 |
| FR-04-04 | 点击网格单元格可弹出该区域的详细识别数据（虫害类型、置信度、时间戳） | 弹窗数据与数据库一致 |
| FR-04-05 | 支持"病害视图"与"虫害视图"一键切换 | 切换后热力图数据源立即更新 |
| FR-04-06 | 支持多维度交叉热力分析（同时展示病害+虫害的复合热力层） | 复合层支持透明度调节 |
| FR-04-07 | 热力图图层与视频流画中画模式叠加 | 可在热力图上叠加实时视频帧 |

#### 3.5.3 输入/处理/输出

- **输入**：各网格区域的 CV 推理结果（类型、置信度、时间戳）、网格空间配置（FR-01）
- **处理**：前端 CSS3D 空间变换 + ECharts 热力图图层复合渲染
- **输出**：2.5D 交互式时空热力图

#### 3.5.4 工业/智能体现点

> 将 CV 数据与空间网格结合，以颜色深浅演变展示虫害扩散趋势，具备"数字孪生"雏形。用户无需逐个查看摄像头画面，即可快速锁定受灾最严重的物理位置。

### 3.6 FR-05：动态巡检排班与定时推理流水线

#### 3.6.1 功能描述

后端定时器驱动机位模拟抓拍与 CV 推理，实现无人值守的自动轮询检测，无需人工盯着屏幕。

#### 3.6.2 功能详细需求

| 子编号 | 需求描述 | 验收标准 |
|--------|---------|---------|
| FR-06-01 | 支持配置巡检排班计划：巡检间隔（分钟级）、巡检机位列表、巡检时段 | 排班计划可增删改查 |
| FR-06-02 | 定时任务调度器按排班计划自动触发摄像头抓拍 | 抓拍时间误差 < 5 秒 |
| FR-06-03 | 抓拍图片自动送入异构双推理流水线（病害模型 + 虫害模型）并行推理 | 两张模型推理可并行执行 |
| FR-06-04 | 推理结果自动入库并实时推送至前端 WebSocket | 推理到前端展示延迟 < 3 秒 |
| FR-06-05 | 支持临时手动触发单次巡检（立即抓拍+推理） | 手动触发不影响正常排班 |
| FR-06-06 | 巡检日志记录每次抓拍时间、机位、推理结果、耗时 | 日志可查询和导出 |
| FR-06-07 | 支持巡检暂停/恢复/跳过某个机位 | 操作即时生效 |

#### 3.6.3 输入/处理/输出

- **输入**：巡检排班配置（CRON 表达式或可视化排班）、摄像头 RTSP 地址
- **处理**：定时器触发 → 抓拍 → 双模型并行推理 → 结果入库 → WebSocket 推送
- **输出**：推理结果记录、巡检日志

#### 3.6.4 工业/智能体现点

> 后端定时器驱动机位模拟抓拍与 CV 推理，无需人工盯着屏幕。实现真正的无人值守自动化巡检。

### 3.7 FR-06：事件驱动型实时智能工单系统

#### 3.7.1 功能描述

突发高置信度灾情时，系统自动异步生成突发工单，通过邮件网关自动外发，打破软件孤立状态，实现专家即时响应。

#### 3.7.2 功能详细需求

| 子编号 | 需求描述 | 验收标准 |
|--------|---------|---------|
| FR-06-01 | 当某次推理结果的置信度超过预设阈值（可配置），自动触发工单生成 | 触发条件可配置 |
| FR-06-02 | 工单自动生成，包含：灾情类型、发生位置（网格编号）、置信度、抓拍图片、时间戳 | 工单信息完整 |
| FR-06-03 | 工单通过邮件网关自动外发至预配置的专家邮箱列表 | 邮件发送延迟 < 30 秒 |
| FR-06-04 | 邮件内容为 HTML 格式，包含现场图片和一键操作按钮 | 邮件在主流邮箱客户端中正常渲染 |
| FR-06-05 | 工单具有完整的状态机：`待处理 → 处理中 → 已完成/已忽略` | 状态转换有完整日志 |
| FR-06-06 | 支持工单升级机制：超过 N 小时未处理自动升级通知更高级别人员 | N 可配置 |

## 8. 用例规约

### 8.1 UC-01：系统自动巡检并生成工单

| 项目 | 描述 |
|------|------|
| 用例名称 | 系统自动巡检并生成工单 |
| 参与者 | 系统（无人值守） |
| 前置条件 | 巡检计划已配置且启用，摄像头在线 |
| 触发条件 | 定时任务到达预设时间 |

**基本流：**

1. 定时调度器触发巡检任务
2. 系统按排班计划依次对各摄像头执行抓拍
3. 抓拍图片送入病害推理流水线和虫害推理流水线并行推理
4. 推理结果合并，按置信度阈值过滤
5. 结果入库，同时通过 WebSocket 推送至前端
6. 如果任一检测结果置信度超过突发阈值（如 0.90）：
   a. 自动生成突发工单
   b. 工单通过邮件网关外发至专家邮箱
7. 前端大屏热力图实时更新

**异常流：**

- 3a. 摄像头抓拍失败 → 记录错误日志，跳过该机位，继续下一个
- 3b. CV 推理服务不可用 → 进入重试队列，最多重试 3 次
- 6b. 邮件发送失败 → 进入邮件重试队列，记录发送失败日志

**后置条件：** 推理结果已入库，前端已更新，工单（如有）已生成

### 8.2 UC-02：专家通过邮件回调处理工单

| 项目 | 描述 |
|------|------|
| 用例名称 | 专家通过邮件回调处理工单 |
| 参与者 | 农技专家 |
| 前置条件 | 系统已发送包含 Token 的工单邮件 |
| 触发条件 | 专家点击邮件中的操作按钮 |

**基本流：**

1. 专家在邮箱中打开工单邮件
2. 专家点击"已确认"按钮（包含回调 Token 的 URL）
3. 浏览器打开 Token 回调页面
4. 后端验证 Token 有效性（未过期、未使用、哈希匹配）
5. 后端更新工单状态为"处理中"
6. 专家在回调页面填写指导意见（可选），点击提交
7. 后端更新工单状态为"已完成"，记录专家指导意见
8. Token 标记为已使用
9. WebSocket 推送工单状态变更至前端大屏
10. 系统发送确认回执邮件给专家

**异常流：**

- 4a. Token 已过期 → 显示"链接已过期，请联系系统管理员"
- 4b. Token 已使用 → 显示"该工单已被处理"
- 4c. Token 无效 → 显示"无效链接"

**后置条件：** 工单状态已更新，大屏已刷新，回执邮件已发送

### 8.3 UC-03：查看 2.5D 热力图并切换视图

| 项目 | 描述 |
|------|------|
| 用例名称 | 查看 2.5D 热力图并切换视图 |
| 参与者 | 农场管理者 / 系统管理员 |
| 前置条件 | 系统已运行一段时间，有推理数据积累 |
| 触发条件 | 用户访问主控大屏 |

**基本流：**

1. 用户登录系统，进入主控大屏
2. 系统加载 2.5D 大棚俯视图（CSS3D 渲染）
3. 各网格单元格根据最近一次推理数据渲染热力颜色
4. 用户点击"切换至虫害视图"按钮
5. 热力图数据源切换为虫害推理流水线数据，颜色重新映射
6. 用户拖动时间轴至昨天 14:00
7. 热力图回放至该时刻的状态
8. 用户点击 Grid-B3 单元格
9. 弹出详情弹窗：该区域虫害类型、置信度、抓拍图片缩略图

**异常流：**

- 3a. 暂无推理数据 → 显示"暂无数据"占位图
- 6a. 请求的历史时间点无数据 → 相邻时间点数据插值显示
