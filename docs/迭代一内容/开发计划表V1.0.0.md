## 七、技术选型概览

**后端技术** — Spring Boot 2.7.x + MyBatis-Plus（责任人：黄博、陆华飞）
> 轻量自动配置、简化CRUD，适配学生开发节奏，社区资源丰富

**数据库** — MySQL 8.0 + Redis缓存（责任人：陆华飞）
> 开源免费、稳定性强，学生熟悉度高，Redis辅助验证码与缓存

**前端技术** — Vue 3 + Vite + Element Plus（责任人：陈嘉豪、陈思宇、程晨）

> 轻量热更新、组件丰富，减少样式开发工作量，适配快速开发

**数据可视化** — ECharts 5.x（责任人：陈嘉豪）

> 图表类型丰富、文档完善，适配数据大屏与统计图表需求

**AI推理服务** — Python 3.10 + FastAPI + ONNX Runtime（责任人：邵建疆、粟昱斌）
> 高性能异步框架，YOLOv8模型推理，与Java业务端物理隔离

**接口文档** — Knife4j 4.x（责任人：黄博）
> 自动生成接口文档，支持在线调试，便于前后端协作

**测试工具** — Postman + JUnit 5（责任人：全体成员）
> 覆盖接口/单元测试场景，操作简单，适合快速掌握

**部署工具** — Docker + Docker Compose（责任人：陆华飞）
> 环境一致性强，部署便捷，减少环境配置冲突

**版本控制** — Git + Gitee/GitHub（责任人：全体开发人员）
> 分布式版本控制，支持团队协作与代码回滚

## 八、风险识别与应对措施

### 8.1 需求风险（风险等级：中）
- **风险描述：** 对用户故事理解偏差，开发功能不符合预期
- **责任人：** 黄博（PO）
- **应对措施：**
  1. 需求分析后与项目指导王犇评审
  2. 绘制流程时序图
  3. 编写需求用例

### 8.2 技术风险（风险等级：中）
- **风险描述：** 对Vue 3/Spring Boot不熟悉，核心功能开发受阻
- **责任人：** 黄博、陆华飞、陈嘉豪、陈思宇、程晨
- **应对措施：**
  1. 前置技术预研培训
  2. 参考若依等开源框架
  3. 及时求助项目指导王犇

### 8.3 进度风险（风险等级：高）
- **风险描述：** 迭代任务堆积，7天周期紧张
- **责任人：** 陈嘉豪（SM）
- **应对措施：**
  1. 每日15分钟站会同步进度
  2. 拆分≤0.5天的小任务
  3. 预留缓冲时间

### 8.4 算法风险（风险等级：中）
- **风险描述：** YOLOv8模型集成困难，推理服务不稳定
- **责任人：** 邵建疆、粟昱斌
- **应对措施：**
  1. 提前准备ONNX模型文件
  2. 准备Mock接口兜底
  3. 算法与业务端解耦

### 8.5 团队风险（风险等级：中）
- **风险描述：** 协作不畅，分工不明确导致重复/遗漏任务
- **责任人：** 陈嘉豪（SM）
- **应对措施：**
  1. 明确分工清单
  2. Git分支管理（主分支+迭代分支）
  3. 每日团队站会

### 8.6 环境风险（风险等级：中）
- **风险描述：** 开发环境不一致，部署失败
- **责任人：** 陆华飞
- **应对措施：**
  1. 统一Docker开发环境
  2. 提前测试部署流程
  3. 准备本地备用环境

### 8.7 质量风险（风险等级：中）
- **风险描述：** 测试不充分，核心流程存在Bug
- **责任人：** 全体成员、黄博（PO）
- **应对措施：**
  1. 编写详细测试用例
  2. 迭代间回归测试
  3. 项目指导王犇验收核心功能
