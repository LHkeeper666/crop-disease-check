# 项目评审一：项目启动评审

### 任务描述：

审查项目启动阶段任务的完成情况。

### 任务指导：

1. 本阶段分值占比：20%
2. 评审项及评审依据：

| 序号 | 内容                               | 要求 |
| ---- | ---------------------------------- | ---- |
| 1    | 《团队章程》                       | 必须 |
| 2    | 《需求规格说明书》                 | 必须 |
| 3    | 《技术选型说明文档》               | 必须 |
| 4    | 《开发计划表》                     | 必须 |
| 5    | 远程代码仓库地址（含访问权限说明） | 必须 |
| 6    | 本地开发环境截图（团队互检通过）   | 必须 |
| 7    | 《用户故事清单》                   | 可选 |
| 8    | 《风险识别与管控表》               | 可选 |
| 9    | 其他材料压缩包                     | 可选 |

### 任务一结果：

- 自组织团队名单（Excel 格式）
  - 包含：团队成员姓名、学号、联系方式、拟承担角色

- 团队分工表（Excel 格式）
  - 包含：角色名称、职责描述、担任成员、主要任务

- 《团队章程》（Word 格式，版本号 V1.0.0）
  - 包含：团队名称、成员列表、角色职责、沟通机制、会议规则、代码规范、工具与文档规范、冲突解决机制、签名页

### 任务二结果：

- 《需求规格说明书》
- 用户故事
- 用户故事地图
- 产品待办列表（基线为 DOCX 格式，动态更新为 Excel 格式）
- 需求评审记录
- CodeArts 思维导图规划（截图）
- CodeArts 甘特规划（截图）

### 任务三结果：

- 技术选型说明文档
- 风险识别与管控文档（含风险类型、风险描述、应对措施、责任人）
- 开发计划表（含迭代周期、阶段任务、时间节点、责任人、交付物）

### 任务四结果：

- 远程代码仓库地址（含访问权限说明）
- 本地开发环境（截图，团队互检）
- 分支管理规范





# 项目评审二：迭代 1 评审

### 任务描述

对第一次迭代（MVP 开发）的成果进行评审，评估敏捷协作流程的可行性与 MVP 功能的完整性。

1. 本阶段分值占比：20%
2. 评审项及评审依据：

| 序号 | 内容                         | 要求 |
| ---- | :--------------------------- | ---- |
| 1    | 冲刺计划会议纪要             | 必须 |
| 2    | 冲刺待办列表（基线）         | 必须 |
| 3    | 冲刺成果演示录屏（3-5 分钟） | 必须 |
| 4    | 轻量化用户手册 V1.0          | 必须 |
| 5    | Git 提交记录                 | 必须 |
| 6    | 每日站会记录（按日期整理）   | 必须 |
| 7    | 代码审查记录                 | 可选 |
| 8    | 缺陷跟踪记录表               | 可选 |
| 9    | 迭代回顾会议纪要             | 可选 |
| 10   | 其他材料压缩包               | 可选 |

### 任务五结果

- 冲刺计划会议纪要
- 冲刺待办列表（待办基线，DOCX 格式）
- 冲刺待办列表跟踪表（在线文档，含任务 ID、标题、负责人、计划日期、实际日期、状态、备注等字段）
- 冲刺目标文档
- 在 CodeArts 中创建的迭代视图（截图）

### 任务六结果

- 每日站会记录（按日期整理）
- 障碍清单（含障碍描述、责任人、解决措施、解决状态）
- 燃尽图 / 看板进度视图（截图）
- 轻量测试用例
- 测试报告
- 缺陷跟踪记录表
- Git 提交记录
- 代码审查记录

### 任务七结果

- 冲刺评审会议纪要
- 用户变更记录表
- 评审反馈表（含反馈人、反馈内容、类型 (缺陷 / 优化 / 新需求)、处理意见）

### 任务八结果

- 冲刺成果演示记录
- 冲刺成果演示录屏（MP4 格式，3-5 分钟）
- 程序源码（压缩包，含完整项目代码）
- 轻量化用户手册
- 交付确认单

### 任务九结果

- 冲刺回顾会议纪要
- 改进计划表（含改进项、现状描述、改进目标、具体措施、责任人、计划完成时间、验证标准）
- 更新版《团队章程》（如有需要）
