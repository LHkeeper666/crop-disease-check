# 一、前言

## 1.1 编写目的

本手册为智慧农业病虫害监测系统迭代2专用操作指南，聚焦迭代2新增与优化的核心功能，包括检测阈值联动、摄像头逻辑修正、农情报表完善、AI Agent专业技能、日报Word导出等，帮助用户快速掌握系统最新操作。

## 1.2 适用范围

仅适用于智慧农业病虫害监测系统迭代2已交付功能，供系统管理员、农业专家、农场管理者使用。文档不包含系统部署与初始配置，相关需求可联系技术支持部门。

## 1.3 读者对象

- 系统管理员：负责用户管理、系统配置、权限分配、摄像头管理
- 农业专家：负责审核上报、制定防治方案、工单处理、AI对话
- 农场管理者：负责查看报表、监控环境数据、处理工单、导出报告

由绿盾先锋项目组起草、维护与解释。

# 二、产品简介

## 2.1 系统定位

迭代2在迭代一基础上，重点完善了检测阈值与工单联动逻辑、修正了摄像头区域配置、优化了农情报表数据源、配置了AI Agent专业技能，并新增了日报Word导出功能。系统功能更加完善，用户体验显著提升。

## 2.2 核心功能（迭代2新增/优化）

- 检测阈值联动：置信度阈值判定、重复工单检测、四处联动展示
- 摄像头逻辑修正：A/B/C三区各3个摄像头，状态与实际一致
- 农情报表完善：数据源修正、日报生成逻辑优化、七日趋势连续展示
- AI Agent技能：农学家、病虫害学家、植物学家三个专家角色
- 日报导出：支持导出.docx格式Word文档
- 页面优化：修复页面切换白屏、设备管理刷新等问题

## 2.3 应用场景

适配温室大棚病虫害监测需求，支持病虫害识别、工单处理、数据统计、AI专家对话等核心业务场景，可直接用于农业企业内部管理。

# 三、术语与定义

| 术语名称 | 定义说明 |
|----------|----------|
| JWT | JSON Web Token，用于用户认证和授权的令牌机制 |
| SSE | Server-Sent Events，服务器发送事件，用于流式数据传输 |
| YOLOv8 | 当前行业领先的实时目标检测算法，用于病虫害识别 |
| MINIO | 开源对象存储服务，用于存储图片等非结构化数据 |
| 工单 | 系统自动生成的病虫害处理单，用于跟踪处理进度 |
| 热力图 | 以颜色深浅表示风险等级的可视化图表 |
| 置信度 | AI识别结果的可信程度，以百分比表示 |
| 检测阈值 | 触发工单生成的最低置信度值，低于阈值的结果仅保留历史记录 |
| Tool Calling | AI Agent调用外部工具（如查询数据库）的能力 |

# 四、系统登录与登出

## 4.1 系统登录

在浏览器地址栏输入系统访问地址（如http://localhost:3000/）进入登录界面，输入正确的用户名和密码后点击登录按钮即可进入系统。新用户可点击"注册"按钮进行注册，填写用户名、密码、姓名、手机号、邮箱等信息，系统将发送邮箱验证码进行验证，注册成功后等待管理员审核。

注册时可输入企业邀请码加入对应企业。邀请码由系统管理员在"设备管理-用户管理"页面生成。

## 4.2 系统登出

点击左侧导航栏底部的用户头像区域，在弹出的菜单中选择"退出登录"按钮即可安全退出系统。

# 五、遥测总览（Dashboard）

## 5.1 页面布局

遥测总览页面采用三栏布局：左侧为环境数据与报警信息，中间为2.5D交互式热力图，右侧为生长指标与摄像头监控。

## 5.2 环境数据面板

左上角显示四宫格环境数据：空气温度、土壤含水量、环境湿度、光照强度。管理员可双击数值进行编辑，修改后自动保存。

## 5.3 检测阈值设置（迭代2新增）

左侧中部新增"检测阈值"滑块，范围0-100%。**仅当检测结果的置信度>=此阈值时，才会自动生成工单**。低于阈值的检测结果仅保留在历史记录中，不会触发工单生成。

调整阈值后，报警面板和热力图会实时联动更新。

## 5.4 2.5D热力图

中间区域为2.5D交互式热力图，展示各网格区域的风险等级。热力图数据来源于工单：有活跃工单的网格区域会显示对应颜色（红=严重、橙=高、黄=中、低=绿）。

- 自动旋转：热力图每30秒自动旋转360度
- 手动拖拽：鼠标拖拽可自由旋转视角
- 重置视角：点击重置按钮恢复默认视角
- 网格详情：点击网格单元格可查看详细信息

**迭代2优化**：热力图数据源与工单逻辑完全匹配，确保数据一致性。

## 5.5 报警面板

左侧底部显示实时报警列表，按置信度排序。仅显示置信度>=检测阈值的活跃工单（待处理+处理中状态）。

## 5.6 七日趋势图（迭代2优化）

右下角显示七日病虫害趋势堆叠柱状图，数据来源于后端统计接口。**迭代2优化**：补充了历史日期数据点，展示连续7天的折线走势，而非仅当天的两个散点。

## 5.7 摄像头监控（迭代2优化）

右侧底部显示摄像头实时监控缩略图。**迭代2优化**：修正为A/B/C三区各3个摄像头共9个，摄像头状态（在线/故障/未连接）与实际硬件一致。

# 六、智能工单（WorkOrders）

## 6.1 工单列表

页面顶部显示统计卡片：总工单数、待处理、处理中、已完成。支持按状态、严重程度、日期范围筛选。

## 6.2 工单状态规范（迭代2优化）

**迭代2统一工单为四种状态**：

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| 待处理 | 新工单，等待指派 | 确认处理、忽略 |
| 处理中 | 已接手，正在处理 | 标记完成 |
| 已完成 | 问题已解决 | 无 |
| 忽略 | 评估后不处理 | 无 |

## 6.3 检测阈值联动（迭代2新增）

当图像检测或摄像头实时检测的结果置信度>=前端设置的检测阈值时，系统会自动生成工单，并在以下四处同步展示：

1. **遥测总览**：热力图更新 + 报警面板新增
2. **智能工单**：工单列表新增（状态为"待处理"）
3. **农情报表**：七日趋势 + 统计报表更新
4. **实时监控**：摄像头检测结果展示

## 6.4 重复工单判定（迭代2新增）

- **同一地点 + 相同问题**：不重复生成工单，避免工单堆积
- **不同地点 + 相同问题**：各自独立生成工单，分别跟踪处理

## 6.5 工单详情

点击工单卡片可查看详情，包括：工单信息、状态流转历史、操作按钮。支持的操作：确认处理、标记完成、忽略、提升/降低严重程度、删除。

## 6.6 创建工单

点击"创建工单"按钮可手动创建工单，填写标题、网格区域、病虫害类型、严重程度、指派专家、置信度等信息。

## 6.7 邮件通知

点击工单详情中的"发送邮件"按钮，可向专家发送邮件通知。AI Agent会自动撰写邮件内容，用户可选择收件人后发送。

\newpage

# 七、图像检测（Detection）

## 7.1 图片上传

支持拖拽或点击上传图片，格式支持JPG/PNG，单张最大10MB，最多同时上传20张。

**迭代2优化**：上传后图片显示区域大小固定，"开始检测"按钮不会被推出可视区域。

## 7.2 检测参数

- **置信度阈值**：滑块调节，范围0.1-0.9，默认0.5
- **网格区域**：选择检测对应的网格区域（A1-C3）

## 7.3 单张检测

上传单张图片后点击"开始检测"，系统调用AI推理服务进行病虫害识别，返回标注图、病害检测列表、虫害检测列表、推理耗时等信息。

## 7.4 批量检测

上传多张图片后点击"开始批量检测"，系统依次对每张图片进行推理。支持缩略图导航切换查看各张图片的检测结果。

## 7.5 自动工单生成

检测完成后，系统自动检查每个检测结果：若置信度>=检测阈值且无重复工单，则自动创建工单（严重程度根据置信度自动分类）。

\newpage

# 八、实时监控（CameraMonitor）

## 8.1 摄像头列表

**迭代2优化**：修正为A/B/C三区各3个摄像头共9个。左侧设备列表显示每个摄像头的检测次数徽章。

## 8.2 监控画面

支持1/2/3列布局切换。每个摄像头显示MJPEG实时画面，通过WebSocket接收推理服务的检测结果，在画面上叠加绘制检测框（红色=病害，蓝色=虫害）。

## 8.3 摄像头状态（迭代2优化）

摄像头状态与实际硬件一致：
- **在线**：正常连接，可查看实时画面
- **故障**：设备异常，无法获取画面
- **未连接**：设备未接入系统

**迭代2优化**：Dashboard、设备管理、实时监控三处的摄像头状态保持一致。

## 8.4 检测详情

点击摄像头可查看详情：摄像头名称、状态、覆盖网格、实时检测结果（病害列表、虫害列表）。

\newpage

# 九、设备管理（DeviceManagement）

## 9.1 摄像头管理

支持摄像头的增删改查。编辑弹窗包含：名称、RTSP主/子流URL、经纬度、方向角、覆盖网格、抓拍分辨率（320/640/1280）、JPEG质量、重连间隔等参数。

## 9.2 网格区域管理

以卡片形式展示各网格区域的标签、作物类型、面积。管理员可编辑标签、作物类型、关联温室等信息。

## 9.3 用户管理（管理员专属）

**迭代2修复**：修复了admin刷新浏览器后用户管理模块无内容的问题。

支持用户的增删改查，包括：搜索、编辑（姓名、角色、手机、邮箱）、重置密码（生成随机8位密码并显示一次）、启用/禁用状态切换。

\newpage

# 十、农情报表（Reports）

## 10.1 统计卡片

页面顶部显示五个统计卡片：总报告数、今日报告、待审核、已处理、高风险预警。

## 10.2 病害/虫害分布饼图（迭代2优化）

左侧显示病害分布和虫害分布两个饼图。**迭代2优化**：数据来源修正为当日未被忽略的工单（待处理+处理中+已完成三种状态），忽略状态不参与统计。

## 10.3 七日趋势折线图（迭代2优化）

右侧上方显示七日病虫害趋势折线图，包含病害和虫害两条曲线。**迭代2优化**：补充了历史日期数据点，展示连续7天的趋势走势。

## 10.4 每日报告表格

右侧下方显示每日报告表格，包含：日期、检测数、病害数、虫害数、处理率、生成状态等列。

## 10.5 导出Excel

点击"导出Excel"按钮，可将当前统计数据导出为.xlsx格式文件。

## 10.6 生成日报（迭代2优化）

**迭代2优化**：
- 生成日报后数据大屏内容保留，不会被清空
- 未接入Agent API时按钮置灰，提示需先配置
- 接入API后可生成包含完整数据分析的文档

## 10.7 导出Word（迭代2新增）

点击"导出Word"按钮，可生成.docx格式的日报文档，包含：
- 数据概览（总检测数、病害数、虫害数、处理率）
- 病害分布表格
- 虫害分布表格
- 每日报告明细表格

# 十一、智慧大脑（Agent）

## 11.1 AI对话

采用SSE流式对话，消息逐字显示。支持Markdown渲染，包括代码高亮、表格、列表等格式。

## 11.2 工具调用可视化（迭代2优化）

AI Agent可调用以下工具，调用过程在界面上实时展示：

| 工具名称 | 功能 | 调用时机 |
|----------|------|----------|
| work_order | 查询工单/按状态统计 | 用户询问工单相关问题 |
| environment | 查询环境数据/聚合趋势 | 用户询问温室环境情况 |
| detection | 查询检测记录/趋势 | 用户询问检测历史 |
| pest_disease_info | 搜索病虫害知识库 | 用户询问病虫害防治 |
| create_work_order | 交互式创建工单 | 用户要求创建工单 |

## 11.3 Agent专业技能（迭代2新增）

**迭代2为AI Agent配置了三个专家角色**：

- **农学家**：作物种植管理、施肥方案、灌溉策略、土壤改良、设施农业管理
- **病虫害学家**：病虫害识别、防治方案、农药使用规范、安全间隔期、综合防治策略
- **植物学家**：植物生理、生长周期、品种特性、环境胁迫响应、植物激素

Agent会根据用户问题自动匹配对应专家角色，给出专业、准确的农业建议。

## 11.4 会话管理

左侧面板显示历史会话列表，支持切换会话、新建会话。每个会话独立保存消息历史。

## 11.5 API设置

右上角设置面板可配置：供应商（DeepSeek/小米MiMo）、模型选择、API密钥（可选，不填则使用后端默认）、验证与保存。

## 11.6 预设问题

页面底部提供7个可点击的快速提问，帮助用户快速了解Agent能力。

# 十二、病虫害图鉴（Handbook）

## 12.1 病害图鉴

展示38种植物病害信息卡片，每张卡片包含：病害图片、病害名称、作物名称、健康/病害状态标签。支持按名称或作物搜索。

## 12.2 虫害图鉴

展示102种虫害信息卡片，每张卡片包含：虫害图片、虫害名称、"虫害"标签。支持按名称搜索。

## 12.3 卡片交互

鼠标悬停时卡片具有3D倾斜效果，图片支持懒加载优化性能。
