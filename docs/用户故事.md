# 2. 项目概述

## 2.1. 项目背景

当前农场病虫害识别主要依赖人工经验判断，存在识别效率低、误判率高、数据难以沉淀与追溯等痛点：农场员工需要逐株排查作物，工作负担重；农技员难以快速获取全局病虫害态势，决策滞后；历史识别数据缺乏系统化管理，无法支撑模型优化与统计分析。

本植物病虫害识别系统定位为"从图片采集到识别反馈再到防治方案的全流程智能化工具"，通过图像识别技术与Web应用结合，覆盖图片上传、病虫害识别、结果审核、防治方案制定、数据统计等核心业务环节，实现病虫害监测的数字化与智能化。

## 2.2. 项目目标

### 2.2.1. **业务目标**

* 提升病虫害识别效率，将人工识别时间从平均5分钟/张缩短至系统识别≤3秒/张；
* 降低误判率，通过模型置信度标记与农技员复核机制，确保最终结果准确率≥90%；
* 实现病虫害数据的结构化沉淀，支撑统计分析与模型持续优化。

### 2.2.2. **功能目标**

* 完成"图片上传→病虫害识别→结果展示→审核与防治方案→数据导出"核心流程的功能闭环；
* 实现数据大屏可视化，支持农场负责人直观查看病虫害总体态势；
* 实现系统管理功能，支持作物与病虫害数据的维护及操作日志审计。

### 2.2.3. **非功能目标**

* 系统核心功能响应时间≤3秒，支持图片批量上传（单次≤10张）；
* 图片上传大小限制≤10MB/张，支持JPG、PNG格式；
* 适配PC端主流浏览器（Chrome、Edge），保障操作兼容性。

## 2.3. 核心业务场景

本系统覆盖以下3个核心端到端业务场景，各场景对应后续用户故事模块：

* 病虫害上报与识别场景

流程：员工上传图片→填写基本信息→系统自动识别→展示识别结果与置信度

对应模块：图像上报模块、病虫害识别模块

* 审核与防治场景

流程：农技员查看待审核记录→复核低置信度结果→制定防治方案→反馈至员工

对应模块：审核与防治方案模块

* 数据统计与管理场景

流程：识别数据积累→统计分析→数据导出→数据大屏展示→系统数据维护

对应模块：数据统计与导出模块、数据大屏模块、系统管理模块

# 3. 用户故事管理规则

## 3.1. 用户故事优先级定义

优先级用于明确用户故事的开发顺序，按"业务价值-紧迫性-依赖关系"三维度判定，分为P0、P1、P2三级：

* P0（最高优先级）：核心业务流程必需，不实现会导致后续流程阻塞。例如"图片上传与病虫害识别""审核与方案制定"等，需优先纳入迭代开发。
* P1（中优先级）：优化业务效率，存在临时替代方案，不影响核心流程闭环。例如"置信度低内容标记""操作日志查看"等，可按迭代容量灵活安排。
* P2（低优先级）：非核心补充功能，仅提升使用便捷性，可延后至后续迭代。例如"数据库管理功能"等，优先保障P0、P1故事交付后再规划。

优先级判定需经产品负责人、技术负责人共识，在用户故事文档中明确标注，若需求变更需重新评估优先级并同步更新。

## 3.2. 故事点估算标准

故事点是衡量用户故事开发复杂度的相对单位，采用斐波那契数列（1、2、3、5、8）作为估算刻度，具体标准如下：

* 1个故事点：简单查询或展示类功能，逻辑单一无依赖。
* 2个故事点：简单新增/修改类功能，无跨模块依赖。
* 3个故事点：单模块内完整流程功能，含基础校验逻辑。
* 5个故事点：跨模块关联功能，含复杂业务逻辑或接口交互。
* 8个故事点：多模块联动或复杂逻辑功能，需技术方案论证。

## 3.3. 验收标准编写规范

验收标准（AC）是判定用户故事是否"完成"的核心依据，需符合"可测试、无歧义、全覆盖"原则，统一采用"Given-When-Then"场景化格式：

* 格式规范：每个验收标准需明确初始状态（Given）、用户操作（When）、预期结果（Then）。
* 内容要求：需覆盖正常场景、边界场景、异常场景。
* 量化指标：涉及性能的需求需明确量化标准。
