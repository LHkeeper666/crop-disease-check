# 悬浮球聊天接口规范

## 1. 概述

悬浮球聊天功能允许用户在任意页面唤起 AI 聊天窗口，AI 自动感知当前页面上下文。

**核心思路**：
- 前端传递「用户在看什么」（selectedId + visibleData）
- 后端根据 selectedId 查询完整数据，合并 visibleData 后注入 AI prompt

## 2. 接口定义

### 2.1 聊天接口（扩展）

**请求**：`POST /api/agri-brain/chat`

**请求体**：
```typescript
interface ChatRequest {
  message: string;           // 用户消息
  conversationId?: string;   // 对话ID，为空则新建
  context?: PageContext;     // 页面上下文（可选）
}
```

### 2.2 PageContext 定义

```typescript
interface PageContext {
  page: string;              // 路由路径，如 "/workorders"
  pageName: string;          // 页面名称，如 "工单管理"
  selectedId?: string;       // 当前选中的资源 ID（可选）
  visibleData?: VisibleData; // 页面可见数据摘要（可选）
}
```

### 2.3 VisibleData 定义

```typescript
interface VisibleData {
  list?: Record<string, any>[];      // 列表数据（前 N 条，建议最多 5 条）
  stats?: Record<string, any>;       // 统计数据（关键指标）
  filters?: Record<string, string>;  // 当前筛选条件
  extra?: Record<string, any>;       // 其他页面特定数据
}
```

## 3. 前端上下文提取规范

### 3.1 注册机制

每个页面通过 `provide` 注册上下文提取函数：

```typescript
// 在页面组件中
import { provide } from 'vue';

const extractContext = (): PageContext => ({
  page: '/workorders',
  pageName: '工单管理',
  selectedId: selectedWorkOrderId.value || undefined,
  visibleData: {
    list: workorderStore.workorders.slice(0, 5).map(formatWorkOrder),
    stats: {
      total: workorderStore.total,
      pending: workorderStore.pendingCount,
      critical: workorderStore.criticalCount
    },
    filters: {
      status: workorderStore.filterStatus
    }
  }
});

provide('pageContextExtractor', extractContext);
```

### 3.2 悬浮球调用

```typescript

      conversationId: currentConversationId.value,
```

### 4.1 工单管理页面 `/workorders`

```typescript
const extractContext = (): PageContext => ({

  pageName: '工单管理',

      id: wo.id,
      title: wo.title,
      status: wo.status,           // pending / processing / completed
      severity: wo.severity,       // low / medium / high / critical
      pestName: wo.pestName,
      gridLabel: wo.gridLabel,
      createdAt: wo.createdAt
    })),
    // 统计数据
    stats: {
      total: workorderStore.total,
      pending: workorderStore.pendingCount,
      processing: workorderStore.processingCount,
      critical: workorderStore.criticalCount
    },
    // 当前筛选条件

      status: workorderStore.filterStatus || 'all',
      severity: workorderStore.filterSeverity || 'all'
    }
  }
});
```

**典型场景**：
- 用户选中工单 #42，问"这个工单怎么处理" → selectedId=42，后端查完整详情
- 用户问"当前页面哪个工单最优先" → visibleData.list 包含前 5 条，AI 可判断 severity=critical 的最优先

## 5. 后端处理流程

### 5.1 ContextBuilder 逻辑

```java
public String buildContext(PageContext context, String userId, String companyId) {
    if (context == null) return "";

    StringBuilder sb = new StringBuilder();
    sb.append("当前页面: ").append(context.getPageName()).append("\n\n");

    // 1. 处理 selectedId（查数据库获取完整数据）
    if (context.getSelectedId() != null) {
        String detail = querySelectedDetail(
            context.getPage(),
            context.getSelectedId(),
            companyId  // 用于数据隔离
        );
        if (detail != null) {
            sb.append("## 用户当前选中的资源\n");
            sb.append(detail).append("\n\n");
        }
    }

    // 2. 处理 visibleData（直接格式化，不需要查询）
    if (context.getVisibleData() != null) {
        VisibleData vd = context.getVisibleData();

        if (vd.getStats() != null && !vd.getStats().isEmpty()) {
            sb.append("## 页面统计数据\n");
            sb.append(toJson(vd.getStats())).append("\n\n");
        }

        if (vd.getList() != null && !vd.getList().isEmpty()) {
            sb.append("## 当前可见列表（前")
              .append(vd.getList().size()).append("条）\n");
            sb.append(toJson(vd.getList())).append("\n\n");
        }

        if (vd.getFilters() != null && !vd.getFilters().isEmpty()) {
            sb.append("## 当前筛选条件\n");
            sb.append(toJson(vd.getFilters())).append("\n\n");
        }

        if (vd.getExtra() != null && !vd.getExtra().isEmpty()) {
            sb.append("## 其他信息\n");
            sb.append(toJson(vd.getExtra())).append("\n\n");
        }
    }

    return sb.toString();
}
```

### 5.2 selectedId 查询示例

```java
private String querySelectedDetail(String page, String selectedId, String companyId) {
    switch (page) {
        case "/workorders":
            WorkOrder wo = workOrderService.getByIdAndCompany(selectedId, companyId);
            return wo != null ? toJson(wo) : null;

        case "/monitor":
            Camera cam = cameraService.getByIdAndCompany(selectedId, companyId);
            return cam != null ? toJson(cam) : null;

        case "/devices":
            Device dev = deviceService.getByIdAndCompany(selectedId, companyId);
            return dev != null ? toJson(dev) : null;

        case "/reports":
            Report report = reportService.getByIdAndCompany(selectedId, companyId);
            return report != null ? toJson(report) : null;

        case "/detection":
            Detection det = detectionService.getByIdAndCompany(selectedId, companyId);
            return det != null ? toJson(det) : null;

        case "/environment":
            Sensor sensor = sensorService.getByIdAndCompany(selectedId, companyId);
            return sensor != null ? toJson(sensor) : null;

        case "/handbook":
            Article article = handbookService.getByIdAndCompany(selectedId, companyId);
            return article != null ? toJson(article) : null;

        default:
            return null;  // 未知页面不查询
    }
}
```

## 7. 注意事项

### 7.1 数据量控制

- **list**：最多 5 条记录，每条只保留关键字段
- **stats**：只传核心指标，不传冗余字段
- **filters**：只传当前生效的筛选条件
- **extra**：控制大小，只传对 AI 有用的信息

### 7.2 角色权限

- **selectedId 查询**：后端传入 companyId，确保只能访问自己公司的数据
- **visibleData**：前端已做过权限过滤，后端直接使用

### 7.3 向后兼容

- `context` 字段可选，不传时行为与之前完全一致
- 一键建议、同步聊天等接口不受影响
